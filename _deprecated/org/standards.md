# Engineering Standards

## Code Quality

- All code must pass linting and type-checking before PR submission
- Test coverage must not decrease from baseline
- RED/GREEN/Refactor cycle required for all implementation work

## Git Conventions

- Feature branches named: `spec/{spec-type}/{spec-name}`
- Commit messages follow Conventional Commits
- Worktrees used for isolated spec execution

## Review Requirements

- Adversarial review required before PR submission
- Risk assessment must be completed for all specs
- Coverage validation against spec success criteria
