# Glossary

| Term | Definition |
|------|-----------|
| **Spec** | A functional specification describing WHAT and WHY for a unit of work |
| **Plan** | A technical implementation plan describing HOW for a spec |
| **Scratch** | Volatile working memory scoped to a single spec (gitignored) |
| **Context Curation** | The process of promoting valuable scratch findings to persistent tiers |
| **Worktree** | An isolated git working directory for parallel spec execution |
| **Adversarial Review** | A review that actively tries to break or find flaws in the implementation |
| **Spec Lifecycle** | The progression: specified → planned → executed → submitted → archived |
