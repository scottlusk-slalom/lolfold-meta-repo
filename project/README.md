# Project Directory

This directory contains project-level context providing the "what" and "why" for all work.

## Suggested Files

| File | Purpose | Template |
|------|---------|----------|
| `product-brief.md` | Stable product context — vision, users, capabilities | `docs/templates/product-brief.md` |
| `project-plan.md` | Increment-specific scope and objectives | `docs/templates/project-plan.md` |
| `engineering-plan.md` | Phased technical migration/build plan | (generated by `/plan-modernization`) |
| `executive-readout.md` | Stakeholder summary of modernization plan | (generated by `/plan-modernization`) |
| `gate-status.yaml` | Spec lifecycle gate tracking | (managed by `/update-gate`) |
| `project-repositories.yaml` | Target repo registry | — |

## Document Hierarchy

```
Product Brief (stable) → Project Plan (per-increment) → Engineering Plan → Specs
```

## Relationships

- `architecture/` — Technical decisions and legacy analysis
- `planning/` — Live operational trackers
- `requirements/` — PRDs and functional requirements
- `specs/` — Individual work items
